目标：从微信（个人号或企业号）的聊天记录中抽取、追踪并自动生成“事件发展‑跟进报告”。
下面给出一套 完整的技术路线 + 操作步骤，可以在本地或云端实现，兼容 Windows、macOS、Linux（WSL）等常见环境。

1️⃣ 数据获取 – 把微信聊天记录导出成结构化文件

2️⃣ 数据清洗 & 标准化
统一时间格式

把每条消息的时间戳统一为 ISO‑8601（2026-03-02T15:34:00+08:00）。
示例（Python）：
from datetime import datetime, timezone, timedelta
ts = int(msg["timestamp"])           # 毫秒或秒
dt = datetime.fromtimestamp(ts/1000, tz=timezone(timedelta(hours=8)))
msg["iso_time"] = dt.isoformat()
过滤非目标对话

按聊天对象（微信号 / 群聊 ID）筛选，只保留 与你（本人） 的对话。
若要聚焦多个人/群，可在配置文件里列出 目标 ID 列表。
合并多媒体信息

图片、语音、文件等在导出时会生成本地路径或 URL。
将路径写入 JSON 并保留 摘要（如图片 OCR、语音转文本）——后面步骤会用到。
去噪

删除系统提示（如 “XXX 加入了群聊”）。
将连续的 同一用户 消息合并为一个 “段落” 方便后续摘要。

3.2 主题与时间线构建
分块：按天或小时划分对话（可用 iso_time）。
主题聚类：对每个时间块的文本跑 TF‑IDF + K‑means 或 sentence‑BERT + HDBSCAN，得到主题簇。
关键句抽取：对每个簇使用 TextRank 取前 2‑3 条句子，作为该时间段的摘要。

4.2 自动构建步骤
步骤	实现方式
事件触发	设定 关键词触发词（如 “合同”“谈判”“进展”“截止”）用 NER 检测；出现时创建新的 event_id。
关联消息	把后续出现的涉及同一关键词且时间差 ≤ 48 h 的消息标记为 同一事件（基于余弦相似度或关键词匹配）。
状态更新	检测情感倾向或动词（如 “完成”“延期”“批准”）更新 status。
生成摘要	使用 GPT‑4‑Turbo / Claude‑3.5（OpenRouter）或本地 LLaMA‑2‑7B‑Chat（离线）调用 summarize，将每个 step 的关键句压缩成 30‑50 字的摘要。
输出报告	将事件 JSON 渲染为 Markdown/Word 模板，自动填充值。
调度	用系统的 cron（crontab -e）或 OpenClaw 的 heartbeat 定时运行脚本（如每日 09:00）生成最新报告并发送至指定渠道（邮件、Feishu 群、Telegram）。

a. extract.py – 读取微信导出 JSON 并统一时间
import json, os, datetime, pytz

def load_messages(dir_path):
    msgs = []
    for fn in os.listdir(dir_path):
        if fn.endswith('.json'):
            with open(os.path.join(dir_path, fn), encoding='utf-8') as f:
                data = json.load(f)
                for m in data['messages']:
                    # 统一时间（秒 → ms）
                    ts = int(m['timestamp'])
                    dt = datetime.datetime.fromtimestamp(ts/1000,
                                                         tz=pytz.timezone('Asia/Shanghai'))
                    m['iso_time'] = dt.isoformat()
                    msgs.append(m)
    return msgs

if __name__ == '__main__':
    msgs = load_messages('data/export')
    with open('data/all_messages.json', 'w', encoding='utf-8') as f:
        json.dump(msgs, f, ensure_ascii=False, indent=2)
b. ner.py – 关键实体抽取 + 事件触发
from transformers import pipeline
import json, re

ner = pipeline("ner", model="shibing624/bert-base-chinese-ner",
               tokenizer="shibing624/bert-base-chinese-ner",
               aggregation_strategy="simple")

TRIGGER_KEYWORDS = ["合同", "项目", "谈判", "进展", "延期", "完成", "安排"]

def is_trigger(text):
    return any(k in text for k in TRIGGER_KEYWORDS)

def process(messages):
    events = {}
    for m in messages:
        txt = m.get('content','')
        if not txt or not is_trigger(txt):
            continue
        ents = ner(txt)
        participants = [e['word'] for e in ents if e['entity_group']=='PER']
        # 简单事件ID：日期+关键词+序号
        ev_id = f"E{m['iso_time'][:10].replace('-','')}-{txt[:4]}"
        if ev_id not in events:
            events[ev_id] = {"title": txt[:20]+"...", "participants": participants,
                             "messages": []}
        events[ev_id]["messages"].append(m)
    return events

if __name__ == '__main__':
    msgs = json.load(open('data/all_messages.json'))
    evs = process(msgs)
    json.dump(evs, open('data/events_raw.json','w',encoding='utf-8'), ensure_ascii=False, indent=2)
c. track.py – 构建时间线、状态、下一步
import json, datetime
from collections import defaultdict

def build_timeline(event):
    # 按天聚合
    buckets = defaultdict(list)
    for m in event["messages"]:
        day = m["iso_time"][:10]
        buckets[day].append(m["content"])
    timeline = []
    for i, (day, texts) in enumerate(sorted(buckets.items())):
        # 简单摘要：拼接前 200 字
        summary = " ".join(texts)[:200]
        timeline.append({
            "step": i+1,
            "time": day,
            "summary": summary,
            "key_messages": [msg["msg_id"] for msg in event["messages"] if msg["iso_time"].startswith(day)]
        })
    return timeline

def infer_status(event):
    # 检测关键动词
    finished = any("完成" in m["content"] for m in event["messages"])
    delayed = any("延期" in m["content"] for m in event["messages"])
    if finished:
        return "已完成"
    if delayed:
        return "延期"
    return "进行中"

def make_report(events):
    reports = {}
    for ev_id, ev in events.items():
        reports[ev_id] = {
            "event_id": ev_id,
            "title": ev["title"],
            "participants": ev["participants"],
            "timeline": build_timeline(ev),
            "status": infer_status(ev),
            "next_actions": []   # 可自行填充或后期人工补充
        }
    return reports

if __name__ == '__main__':
    raw = json.load(open('data/events_raw.json'))
    final = make_report(raw)
    json.dump(final, open('data/events.json','w',encoding='utf-8'), ensure_ascii=False, indent=2)
d. report.py – 渲染 Markdown
import json, datetime

tmpl = """# 事件报告 {title}
**事件编号**: {event_id}
**参与方**: {participants}
**状态**: {status}

## 时间线
{timeline_md}

## 下一步行动
{next_actions}
"""

def timeline_to_md(tl):
    lines = []
    for s in tl:
        lines.append(f"### 步骤 {s['step']} – {s['time']}\n- 摘要: {s['summary']}\n- 关键消息 ID: {', '.join(s['key_messages'])}\n")
    return "\n".join(lines)

def gen():
    evs = json.load(open('data/events.json'))
    for ev in evs.values():
        md = tmpl.format(
            title=ev["title"],
            event_id=ev["event_id"],
            participants=", ".join(ev["participants"]),
            status=ev["status"],
            timeline_md=timeline_to_md(ev["timeline"]),
            next_actions=", ".join(ev["next_actions"]) or "（待补充）"
        )
        fn = f"reports/{ev['event_id']}.md"
        open(fn, "w", encoding="utf-8").write(md)

if __name__ == '__main__':
    gen()

7️⃣ 常见问题 & 调优建议
问题	解决思路
消息量太大（上万条）导致处理慢	- 先 按日期切块（每月一个文件）<br>- 用 SQLite 存储后再查询（SELECT … WHERE time BETWEEN …）<br>- 并行化 NER：multiprocessing.Pool
语音/视频文字转写	使用 OpenAI Whisper（本地模型 openai/whisper-base）或 阿里云语音识别 API，先把音频转换为文字后再纳入分析。
多语言混杂	对每条消息先用 langdetect 判断语言，再分别走对应语言的 NER/摘要模型。
隐私合规	- 所有数据留在本地（不上传服务器）<br>- 若调用云端 LLM，使用 脱敏（人名、手机号、身份证号正则替换）后再发送。
事件关联错误（误判为同一事件）	调整 相似度阈值（余弦相似度 0.75→0.85），并加入 时间窗口（如 48 h）约束。
8️⃣ 小结
导出：使用 WeChatExporter 或官方 API 把聊天记录转成结构化 JSON。
清洗：统一时间、过滤、合并同一用户的连续消息。
信息抽取：用中文 NER、关键词、情感模型识别 事件触发词、参与人、关键时间点。
事件建模：自动生成 event_id，按时间线聚合并更新状态。
摘要/报告：调用本地/云端 LLM（或 TextRank）生成简洁摘要，渲染为 Markdown/Word。
自动化：放入 cron / OpenClaw heartbeat，定时生成并推送至 Feishu
