## 🦞 什么是OpenClaw？

### 定义与定位
**OpenClaw**（原名Moltbot、Clawdbot）是一个**本地优先、多平台、可扩展的个人AI助手框架**。它不是一个简单的聊天机器人，而是一个完整的AI代理生态系统，可以在你的设备上运行，完全控制你的数据和隐私。

### 核心理念
1. **本地优先**：数据留在你的设备上，不依赖云端
2. **多平台集成**：支持微信、飞书、QQ、Telegram等主流IM工具
3. **技能扩展**：通过技能系统无限扩展功能
4. **开源透明**：完全开源，社区驱动发展

### 发展历程
- **起源**：从Clawdbot项目演变而来
- **更名**：经历Moltbot阶段，最终定名OpenClaw
- **现状**：GitHub上239k stars，46.1k forks（截至2026年2月）
- **社区**：Discord社区活跃，超过5700个技能

### 常见问题解决
```markdown
## 问题1：网关连接断开
**症状**: "已断开与网关的连接。device identity required"
**原因**: OpenClaw 2026.2.22-2版本加强了设备身份验证
**解决方案**:
  1. 使用127.0.0.1代替LAN IP
  2. 配置allowInsecureAuth: true
  3. 设备配对和批准（最安全）

## 问题2：WebSocket连接失败
**症状**: 页面能访问但连接断开
**原因**: 腾讯云安全组限制或Nginx配置问题
**解决方案**:
  1. 检查并配置腾讯云安全组
  2. 配置Nginx反向代理（正确处理WebSocket）
  3. 配置HTTPS和SSL证书
```

## 🔒 安全与隐私考量

### 安全优势
1. **数据本地化**：敏感数据不出设备
2. **权限控制**：细粒度的工具访问控制
3. **审计日志**：完整的操作记录
4. **代码审查**：开源透明，社区监督

### 风险与防范
```markdown
## 已知风险及应对

### 风险1：恶意技能
**案例**: 2026年1月27-29日发现28个恶意技能
**防范**:
  - 只从可信来源安装技能
  - 使用`openclaw-sec`安全检查工具
  - 定期更新技能版本

### 风险2：配置泄露
**案例**: API密钥、Token等敏感信息泄露
**防范**:
  - 使用环境变量存储敏感信息
  - 定期轮换密钥
  - 配置访问日志监控

### 风险3：过度权限
**案例**: 技能请求过多系统权限
**防范**:
  - 最小权限原则
  - 沙箱环境运行不可信技能
  - 人工审核高风险操作
```

### 最佳安全实践
```yaml
部署安全:
  网络隔离: 使用内网或VPN
  访问控制: IP白名单、身份验证
  日志审计: 完整操作日志记录
  定期更新: 及时应用安全补丁

数据安全:
  加密存储: 敏感数据加密
  备份策略: 定期数据备份
  清理策略: 定期清理临时文件
  权限管理: 最小必要权限
```
