3.2 抓取文章内容（Web Fetch）
使用 OpenClaw 自带的 web_fetch（或自己写 requests）把每个 URL 转成 Markdown/纯文本。

Python 示例 (scripts/fetch_wechat.py)：
#!/usr/bin/env python3
import os, json, time, pathlib
import requests
from tqdm import tqdm

LINKS_FILE = pathlib.Path('data/wechat_links.txt')
OUT_DIR    = pathlib.Path('data/articles')
OUT_DIR.mkdir(parents=True, exist_ok=True)

def fetch(url):
    # 使用 OpenClaw 的 web-fetch API（如果本机能直接访问）
    api = f"https://r.jina.ai/http?url={url}"   # Jina Reader 免费版，可直接返回 markdown
    r = requests.get(api, timeout=20)
    r.raise_for_status()
    return r.text

for line in tqdm(LINKS_FILE.read_text(encoding='utf-8').splitlines()):
    url = line.strip()
    if not url: continue
    fname = OUT_DIR / (url.split('/')[-1] + '.md')
    if fname.exists(): continue   # 跳过已抓取的
    try:
        txt = fetch(url)
        fname.write_text(txt, encoding='utf-8')
        time.sleep(0.5)          # 防止短时间请求被限速
    except Exception as e:
        print(f"[ERR] {url} → {e}")

抓取完成后，所有文章会保存在 data/articles/*.md。
3.3 文本清洗 & 切块
使用 LangChain（Python）或 Node‑langchain 进行切块。下面是 Python 示例：
from langchain.text_splitter import RecursiveCharacterTextSplitter
from pathlib import Path
import json, sqlite3, os
import numpy as np
from sentence_transformers import SentenceTransformer
import faiss

# 1️⃣ 加载模型
model = SentenceTransformer('BAAI/bge-large-zh')
model.max_seq_length = 512    # 防止超长截断

# 2️⃣ 切块配置
splitter = RecursiveCharacterTextSplitter(chunk_size=500,
chunk_overlap=80,
separators=["\n\n", "\n", " "])
3️⃣ SQLite 初始化（存元数据）
db_path = Path('data/metadata.db')
conn = sqlite3.connect(db_path)
c = conn.cursor()
c.execute("""CREATE TABLE IF NOT EXISTS docs (
id INTEGER PRIMARY KEY,
title TEXT,
source TEXT,
date TEXT,
tags TEXT,
chunk TEXT
)""")
conn.commit()
4️⃣ FAISS 初始化（IP metric, 与 BGE 兼容）
dim = 768
index = faiss.IndexFlatIP(dim)   # 直接内存索引；如数据量 > 10W，可换成 IVF/HNSW
5️⃣ 遍历所有 markdown 文件
doc_id = 0
for md_path in Path('data/articles').glob('*.md'):
raw = md_path.read_text(encoding='utf-8')
# 简单获取标题（第一行 # 标题）和发布时间（如果有 @2026‑03‑15）
title = raw.splitlines()[0].lstrip('# ').strip()
# 假设文件名里带有日期或手动在正文首行写入 日期: 2026-03-15
import re
m = re.search(r'(\d{4}[‑-]\d{2}[‑-]\d{2})', raw)
date = m.group(1) if m else None
chunks = splitter.split_text(raw)
for chunk in chunks:
    vec = model.encode([chunk])[0].astype('float32')
    # normalize for IP (cosine)
    vec = vec / np.linalg.norm(vec)
    # 写入 FAISS
    index.add(np.expand_dims(vec, 0))
    # 写入 SQLite
    c.execute(
        "INSERT INTO docs (id, title, source, date, tags, chunk) VALUES (?,?,?,?,?,?)",
        (doc_id, title, str(md_path), date, json.dumps([]), chunk)
    )
    doc_id += 1

conn.commit()
print(f"✅ 完成：FAISS 向量 {index.ntotal} 条，SQLite 记录 {doc_id} 条")
> **运行**：`python scripts/build_index.py`  
> **提示**：如果文档量上万条（向量 > 100k），推荐使用 **IVF** 或 **HNSW** 索引（把 `IndexFlatIP` 替换为 `IndexIVFFlat` 并先 `train`）。  
### 3.4 查询接口（FastAPI 示例）
把检索、摘要、分析封装成 HTTP 接口，方便在 OpenClaw‑bot‑review 前端调用。
```python
# file: api/query_api.py
from fastapi import FastAPI, Query
import faiss, numpy as np, sqlite3, json, os
from sentence_transformers import SentenceTransformer
from datetime import datetime, timedelta
import openai   # 或 anthropic
app = FastAPI()
model = SentenceTransformer('BAAI/bge-large-zh')
model.max_seq_length = 512
# 加载已生成的 index 与 sqlite
dim = 768
index = faiss.read_index('data/faiss.index')          # 若使用持久化文件
conn = sqlite3.connect('data/metadata.db')
c = conn.cursor()
def embed(text):
    vec = model.encode([text])[0].astype('float32')
    return vec / np.linalg.norm(vec)
@app.get("/search")
def search(
    q: str = Query(..., description="自然语言查询"),
    top_k: int = 10,
    start_date: str = None,
    end_date: str = None
):
    # 1️⃣ 向量检索
    q_vec = embed(q)
    D, I = index.search(np.expand_dims(q_vec, 0), top_k*3)   # 暂取 3× top_k 防止日期过滤后不足
    ids = I[0].tolist()
    # 2️⃣ 根据 SQLite 拉元数据（并过滤日期）
    placeholders = ",".join("?" for _ in ids)
    sql = f"SELECT id, title, source, date, chunk FROM docs WHERE id IN ({placeholders})"
    rows = c.execute(sql, ids).fetchall()
    # 日期过滤（如果提供）
    if start_date or end_date:
        def in_range(row):
            if not row[3]: return False
            d = datetime.strptime(row[3][:10], "%Y-%m-%d")
            if start_date and d < datetime.strptime(start_date, "%Y-%m-%d"):
                return False
            if end_date   and d > datetime.strptime(end_date,   "%Y-%m-%d"):
                return False
            return True
        rows = [r for r in rows if in_range(r)]
    # 3️⃣ 取前 top_k
    rows = rows[:top_k]
    # 4️⃣ 合并文本用于 LLM 摘要
    context = "\n---\n".join([r[4] for r in rows])
    # 5️⃣ 调用 LLM 生成回答（这里用 OpenAI 的 ChatGPT）
    prompt = f"""
你是一个中文写作助理。请根据信息摘要并回答用户的问题。
用户提问：{q}
检索到的相关文章内容（已按相关度排序）：
{context}
请给出：
1. 每篇文章的标题、发布日期、简要要点（每篇 2‑3 行）；
2. 综合总结这些文章的共同结论；
3. 分析截至上述日期（若提供）OpenClaw 在国内的应用阶段（如 “刚起步、试点、成熟”）并给出依据。
"""
    resp = openai.ChatCompletion.create(
        model="gpt-4o-mini",
        messages=[{"role":"user","content":prompt}],
        temperature=0.2,
        max_tokens=1200,
    )
    answer = resp.choices[0].message.content
    return {"answer": answer, "hits": rows}

6️⃣ 常见问题 & 解决方案
症状
可能原因
解决办法
抓取报 403 / 404
公众号文章防爬虫或需登录
使用 requests 加 User-Agent、Cookie；或通过 wechat_selenium 播放登录后使用 page.content 抓取。
FAISS 加载慢
向量数 > 100k，而使用 IndexFlatIP（全内存）
改用 IndexIVFFlat：index = faiss.IndexIVFFlat(dim, nlist=1000, metric=faiss.METRIC_INNER_PRODUCT); index.train(train_vectors); index.add(all_vectors)
检索不到指定日期
SQLite 中 date 字段为空或格式不统一
在 fetch_wechat.py 把文章发布日期写入文件名或 FrontMatter（如 date: 2026-03-15），在 build_index.py 读取并统一 YYYY-MM-DD 格式保存。
LLM 调用超时
发送的上下文太长（超过模型 token 上限）
把 top_k 降到 5‑10，或先在 FAISS 返回 30 条，再在本地做 rerank（交叉‑encoder）挑出最相关的 5 条再喂给 LLM。
查询结果总是同一批
未对向量进行 归一化，导致 IP 度量失真
vec = vec / np.linalg.norm(vec)（代码里已加入）。
