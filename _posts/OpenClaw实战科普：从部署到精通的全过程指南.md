## 🚀 第二章：我的OpenClaw部署历程

### 2.1 硬件选择：从苹果到Windows的转变

**最初想法**：购买二手苹果笔记本（M1 MacBook Pro）
- **预算**：4000-5000元
- **优势**：性能稳定、续航优秀
- **问题**：价格偏高、扩展性有限

**OpenClaw建议**：Windows笔记本 + WSL2
- **推荐配置**：联想小新Pro 14（2023款）
- **具体参数**：AMD R7 7840HS + 32GB内存 + 1TB SSD
- **价格**：3000-3500元（节省40%）
- **优势**：内存翻倍、性能相当、扩展性更好

**实际体验**：Windows + WSL2完美运行OpenClaw，性能完全满足需求。

### 2.2 云服务器部署：腾讯云实战

**需求背景**：需要24小时在线的OpenClaw服务

**OpenClaw提供的解决方案**：

#### 阶段一：服务器购买与配置
1. **选择配置**：2核4G轻量应用服务器（约80元/月）
2. **系统选择**：Ubuntu 22.04 LTS
3. **安全组配置**：开放3000端口（OpenClaw默认端口）

#### 阶段二：OpenClaw安装
```bash
# 安装Node.js
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装OpenClaw
npm install -g openclaw

# 初始化配置
openclaw init
```

#### 阶段三：企业微信集成
1. **创建企业微信应用**：获取AgentId和Secret
2. **配置Webhook**：设置回调地址
3. **测试连接**：验证消息收发

### 2.3 遇到的实际问题及解决方案

#### 问题一：网关连接失败
**错误信息**："已断开与网关的连接。device identity required"

**OpenClaw分析**：2026.2.22-2版本后加强了设备身份验证

**解决方案**：
1. **简单方案**：使用127.0.0.1代替LAN IP
2. **局域网方案**：配置allowInsecureAuth: true
3. **安全方案**：设备配对和批准
4. **生产方案**：使用HTTPS + 域名

#### 问题二：腾讯云WebSocket连接失败
**现象**：页面能访问但连接断开

**OpenClaw诊断**：
1. 腾讯云安全组限制
2. WebSocket协议处理问题
3. 跨域访问限制

**解决方案**：
1. **配置Nginx反向代理**：
```nginx
location / {
    proxy_pass http://localhost:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

2. **配置HTTPS**：使用Let's Encrypt免费证书
3. **调整OpenClaw配置**：设置正确的host和port
